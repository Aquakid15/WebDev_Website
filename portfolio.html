Biology Concept Map

HTML CSS JavaScript

Overview

An interactive, visually stunning concept map designed to help students master biology at the molecular and cellular level. I built this project to solve a real problem I faced: keeping track of 35+ interconnected biology terms while studying for exams.

Key Features

  • Glassmorphism UI: Modern, frosted-glass design aesthetic that makes content pop
  • Interactive Diagrams: Click-to-reveal animations that make learning engaging
  • Responsive Layout: Works seamlessly on desktop and mobile devices
  • Search Functionality: Quickly find specific terms and concepts

Technical Challenges

The biggest challenge was organizing complex biological relationships in a way that's both visually appealing and educationally effective. I solved this by implementing a card-based layout with CSS Grid and using JavaScript to handle smooth transitions between different concept categories.

Sample Code - Interactive Card System:

// Toggle card expansion on click
document.querySelectorAll('.bio-card').forEach(card => {
    card.addEventListener('click', () => {
        card.classList.toggle('expanded');
        updateRelatedConcepts(card.dataset.term);
    });
});

What I Learned

This project taught me the importance of user-centered design. By building something I personally needed, I learned how to balance aesthetics with functionality. I also gained valuable experience with CSS animations and JavaScript event handling, skills I continue to use in every project.

Lyra Expense Tracker

Swift SwiftUI iOS

Overview

Lyra is a native iOS expense tracking application built with Swift and SwiftUI. The app helps users manage their personal finances by tracking spending habits, categorizing expenses, and visualizing financial data through intuitive charts and summaries.

Key Features

  • Expense Logging: Quick and easy entry system for daily transactions
  • Category System: Organize expenses by custom categories (food, entertainment, etc.)
  • Data Visualization: Charts and graphs showing spending patterns over time
  • Native iOS Design: Follows Apple's Human Interface Guidelines for a polished feel

Design Decisions

I chose SwiftUI over UIKit because it allowed me to build a modern, declarative interface more efficiently. The app uses Apple's SF Symbols for icons, ensuring consistency with the iOS ecosystem. I implemented Core Data for persistent storage, allowing users to track their expenses over months and years.

Sample Code - SwiftUI Expense View:

struct ExpenseRow: View {
    let expense: Expense
    
    var body: some View {
        HStack {
            CategoryIcon(expense.category)
            VStack(alignment: .leading) {
                Text(expense.name)
                Text(expense.date, style: .date)
            }
            Spacer()
            Text("$\(expense.amount, specifier: "%.2f")")
        }
    }
}

Challenges Overcome

Learning SwiftUI's state management system was initially challenging. Understanding the difference between @State, @Binding, and @ObservedObject took time, but mastering these concepts made me a much stronger iOS developer. I also had to figure out how to efficiently query and filter Core Data records for the chart views.

Personal Growth

This project marked my transition from web development to native mobile app development. It taught me the importance of platform-specific design patterns and gave me confidence to explore new technologies. Building Lyra showed me that the problem-solving skills I learned in web development transfer beautifully to mobile development.
